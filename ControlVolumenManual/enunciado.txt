Práctica: Control de volumen con detección de manos 
 
Módulo:M6 – Acceso a datos 
 
 
Objetivos 
 
•Implementar detección de manos en tiempo real con MediaPipe y 
OpenCV. 
•Controlar el volumen del sistema (Windows) mediante gestos con 
pycaw. 
•Diseñar la arquitectura de la aplicación siguiendo el patrón MVC 
(Modelo-Vista-Controlador). 
•Integrar una base de datos NoSQL (MongoDB Atlas) para persistir 
sesiones y eventos de volumen. 
•Aplicar el patrón DAO (Data Access Object) para abstraer el acceso a 
datos. 
•Configurar variables de entorno (.env) para credenciales sensibles. 
 
Descripción de la práctica 
 
Desarrollarás una aplicación en Python que permite controlar el 
volumen del sistema operativo mediante gestos de mano captados por 
la webcam. La aplicación detecta la posición de los dedos pulgar e 
índice y, en función de la distancia entre ellos, ajusta el volumen. Solo 
se aplica el cambio cuando el meñique está bajado (gesto intencional). 
 
Además, la aplicación registrará cada sesión de uso y cada cambio de 
volumen en MongoDB Atlas, permitiendo analizar el uso 
posteriormente. 
 
Requisitos técnicos 
 
El alumno debe entregar una aplicación que incluya: 
 
•Webcam funcional para captura de vídeo. 
•OpenCV (cv2) para procesamiento de imagen y visualización. 
•MediaPipe (Hand Landmarker) para detección de 21 puntos por 
mano. 
•pycaw para control del volumen en Windows (API de audio del 
sistema).

Estructura del proyecto 
 
La solución debe organizarse en módulos coherentes: 
 
main.py—Punto de entrada. Coordina cámara, detector, control de 
volumen y DAO. Implementa el bucle principal (Controller). 
 
HandTrackingModule.py—Módulo reutilizable de detección de 
manos con MediaPipe. Devuelve landmarks, bbox y distancia entre 
dedos. 
 
VolumeHandControl.py—Lógica de mapeo distancia-volumen. 
Versión básica o avanzada (meñique para aplicar). 
 
config/settings.py—Carga de configuración desde .env 
(python-dotenv). 
 
models/session.py—Modelo de datos: sesión (inicio, fin, duración). 
 
models/volume_event.py—Modelo de datos: evento de volumen 
(timestamp, volumen anterior/nuevo, distancia dedos). 
 
dao/mongodb_dao.py—DAO con patrón Singleton. Conexión a Atlas, 
inserción de sesiones y eventos de volumen.

Criterios de evaluación 
 
Detección de manos —Funciona correctamente el tracking de 
manos. Se visualiza la barra de volumen y el porcentaje en tiempo 
real. 
 
Control de volumen —La distancia pulgar-índice mapea al volumen 
del sistema. El gesto con meñique bajado aplica el cambio. 
 
Arquitectura MVC y DAO —Código organizado en modelos, vistas 
(OpenCV) y controlador. DAO centraliza el acceso a MongoDB. 
 
Integración MongoDB —Sesiones y eventos de volumen se guardan 
en Atlas. Indicador visual (DB: OK / DB: --) según conexión. 
 
Extras (opcional) 
 
Mejoras adicionales que se valorarán: mejora de rendimiento (frame 
skip, resolución), mejor manejo de errores de conexión, 
documentación (TUTORIAL.md) para otros alumnos, y pruebas con 
distintas condiciones de luz. 
 
Entrega 
 
Se entregará enlace github al proyecto completo (código fuente, 
requirements.txt) y se deberá mostrar el proyecto funcionando 
accediendo a la estructura de datos en MongoDB Atlas.
